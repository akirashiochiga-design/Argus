Réponse courte

Notre frontend est construit avec React, Vite et Tailwind CSS. Le backend est une API Python avec FastAPI. Les données sont stockées dans SQLite à travers SQLModel et SQLAlchemy. Pour l'IA multimodale, nous utilisons Gemini 2.5 Flash par défaut, avec Claude comme alternative. L'orchestration est une machine à états Python, sans framework agentique lourd. Docker assemble l'application et Railway la déploie derrière une URL unique.

Frontend

React

Construit les écrans et met l'interface à jour quand un dossier change : pipeline, Studio, approbations, dashboard et audit.

Vite

Lance rapidement React en développement et compile le frontend en fichiers optimisés pour la production.

Tailwind CSS

Gère le design : couleurs, grilles, espacements, boutons, cartes et états visuels du pipeline.

Backend

Python

Contient la logique métier : orchestration, garanties, calcul déterministe de l'indemnité, validation humaine, appels IA et audit.

FastAPI

Expose cette logique sous forme d'API HTTP appelée par React. Il valide aussi les données entrantes.

Uvicorn

Est le serveur qui fait tourner FastAPI et répond aux requêtes.

Données

SQLite

Stocke les agents, workflows, dossiers, exécutions, tâches humaines et événements d'audit dans un fichier. C'est un choix adapté au prototype ; PostgreSQL serait la cible pour une production concurrente et multi-tenant.

SQLModel et SQLAlchemy

SQLModel représente les tables avec des classes Python. SQLAlchemy gère les connexions et traduit les opérations Python en requêtes SQL.

Connecteur assurance

Lit une base externe en lecture seule, vérifie son schéma et synchronise polices, garanties et sinistres sans modifier le système source.

Intelligence artificielle

Gemini 2.5 Flash

Lit les déclarations en français ou en darija, extrait les documents, analyse les images et aide à rédiger les courriers.

Claude

Est supporté comme fournisseur alternatif, ce qui évite de dépendre d'un seul modèle.

Limite fondamentale

L'IA ne calcule jamais l'indemnité et ne valide jamais un paiement. Les montants viennent de fonctions Python déterministes et passent obligatoirement par un humain.

Pillow

Prépare les images avant leur analyse par le modèle multimodal.

Orchestration et gouvernance

Machine à états maison

Un workflow est une liste ordonnée d'étapes. Chaque appel exécute une étape, enregistre sa sortie et fait avancer le dossier. C'est plus simple à lire, tester et auditer qu'un framework agentique lourd.

Human-in-the-loop

La validation humaine est une porte bloquante du backend, pas seulement un bouton dans l'interface. Aucun montant ne devient validé sans décision explicite d'un gestionnaire.

Audit append-only

Chaque action crée un événement horodaté avec acteur, objet, avant, après et motif. L'API ne permet pas de modifier ou supprimer ces événements.

Déploiement

Docker

Compile d'abord React avec Node, puis installe Python et FastAPI dans une image unique et reproductible.

Railway

Construit l'image Docker, lance le service, contrôle la route de santé et expose l'application par une URL.

Flux complet

React envoie une requête à FastAPI. FastAPI charge le dossier depuis SQLite avec SQLModel. L'orchestrateur exécute l'étape courante. Le LLM peut lire du contenu non structuré, mais les garanties et montants sont calculés en Python. Le résultat et l'audit sont enregistrés, puis React actualise l'écran. À la porte humaine, tout s'arrête jusqu'à la décision du gestionnaire.
